TASK 019 - MiMC parameter parity guard

Mission:
Ensure MiMC parameters stay aligned with Dark Forest v0.6 and never silently drift.

Background:
Changing MiMC rounds/exponent or using a different MiMC implementation breaks parity for location IDs and artifacts.

Scope:
Contract tests or invariant checks.

Plan:
1) Add an explicit test that validates MiMC outputs for a small fixed set of inputs.
2) Guard against parameter changes by failing if outputs differ.

Acceptance criteria:
- A test fails if MiMC parameters or implementation change.
- No changes to runtime code are required.

Risks / side-effects:
- None; this is a regression guard.

Testing / benchmarks:
- Run MiMC parity test in CI or local test suite.
